Release Notes¶

Version 3.0.0 (Current)¶

Release date: May. 2023

Highlights

  1. Revised Interface, added new Tools section and moved some existing applications to that pulldown

  2. Hazard Occurance Model added to earthquake selection

  3. Revised input file format in anticipation of multiple asset types with multiple options for each

Version 2.2.0¶

Release date: Sept 2022

Highlights

  1. Obtain information from Census and ARC

  2. Updated ground motion input options for OpenQuake

Version 2.1.0¶

Release date: March 2022

Highlights

  1. Ability to perform regional site response analysis taking motions at rock to surface

  2. Ability to import hazards as a raster

  3. Ability to employ custom fragility functions in Pelicun DL framework

  4. Support of tsunami hazards

  5. Ability to import population demographics from census and merge with building inventory

Version 2.0.0¶

Release date: Dec. 2021

  1. QGIS integrated into R2D with the ability to:

    1. load several types of GIS files into R2D to visualize and extract information

post-process results based on advanced queries and expression filters join external databases to GIS features to add additional information for pre- and post-processing export GIS layers into many supported formats for viewing in other GIS applications

  1. Filtering of assets to include in an analysis based on advanced queries and expressions

  2. OpenQuake source selection tool

    1. OpenQuake Scenario-Based Seismic Hazard Analysis to calculate intensity measure from OpenQuake rupture sources

    2. OpenQuake Classical Probabilistic Seismic Hazard Analysis to calculate intensity measures from OpenQuake seismic source and ground motion logic trees

  3. Widget to import building inventory in ShapeFile format

version updates to 2.0.0 as older versions run at design-safe will not load due to changes in backend.

Version 1.0.0¶

Initial Release allows assessment of impact of earthquake on a region

  1. Ground motion Selection using EarthquakeScenario application to obtain scenario specific ground motions for user defined scenario utiling PEER NGA & user defined inputs

  2. Building Inventory input by user

  3. Building Modeling utilizing MDOF-LU

  4. Building Loss Estimation using Pelicun and Hazus fragilities & consequence function.

  5. Interface incorporated ArcGis
